XML file structure

The structure of the XML data file embodies left-to-right streaming flows of values between piles of boxes. Flows in other directions are not supported. The <root> element contains two elements: <piles> for stacks of rectangular boxes and <flows> for the values flowing in between. The example diagram contains two stacks in <piles>, a red and a green one. The colours are assigned automatically, but can be adjusted. Each <pile> consist of two <box>es. The height of the boxes gets determined by the values to be displayed defined in the <flow> elements. The box only needs one attribute <box id=""> with unique content. There can be any number of <pile> stacks and each <pile> can contain any number of <box>es. It is recommended not to have too many small values. In that case, summarize data for a meaningful presentation.

The connection of the boxes through data values is defined in the element <flows>. Inside this element, one <flow> is created for every connection between two boxes. It defines which data value it represents (attribute value=""), which box it starts with (start box id value in the attribute from="") and at which it ends (end box id value in the attribute to="").

The respective <box> does not contain its own information about the data values. Its height is only determined by the flows connected to it. The connection is established through the id attribute of the box.

Text written into the box or flow elements is shown in the graphic. It is also possible to output the values.

In the following table the nodes of the XML tree are listed. Elements are dark red and Attributes are shown in light red in alphabetical order. The text node is also mentioned at the end. He doesn't have a name of his own. The name in the first column is printed bold. The fourth column shows the parent elements the Node can be used in. For the elements, the possible children (nodes contained in the element) are shown. The required children are shows bold, the optional ones are shown normally. Attributes and texts have no child elements.

As a rule, attributes cannot only be used in the element to which they refer affect, but also in the elements above. In that case they affect all elements that are included below and for which they are intended. So the colour can be set for a complete <pile color=""> with just one attribute and a different colour can still be used for a single <box color="">. <root color=""> would set all boxes and flows to the same colour. Despite such a top-level definition different colours can be applied in the levels below, each of which applies to its area.

unit-pattern attribute Pattern for value display. Decimal places, decimal separator and thousands separator are defined here.
Available characters are
as thousands separator: space  , point ., acute ´ and comma ,
as decimal separator: point . and comma ,
as placeholder: Hash # for optional digits and Null 0 for required digits

A number input of 1234567.89 in the XML file can be displayed as follows:

unit-pattern="#.##0" 1.234.568
unit-pattern="#.##0,0" 1.234.567,9

startOffset attribute Horizontal position of the text in a <flow>. Inside the <text> element only permissible if that is located within a <flow> element. The range of values is 0 to 100 and denotes the percentage of the position on the <flow>, where 0 is all the way left and 100 is completely on the right side.

Tip: a text can also be positioned right even though using <flow align="start"> by setting startOffset to a high value like startOffset="75". That way the text ist positioned on the right side controlling the start (left side) of the text.

The default value depends on the value of the attribute align="":
• 1 with align="start"
• 50 with align="middle" and if align is not defined
• 99 with align="end"		 flows

svg:* element In the <root> element any SVG elements be included. These are copied into the generated SVG file without changes. A good knowledge of the SVG standard is helpful.
